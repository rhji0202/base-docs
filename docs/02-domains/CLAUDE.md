# Domains 폴더 가이드 (DDD)

이 폴더는 **비즈니스 로직의 본질**을 담습니다.
도메인 주도 설계(DDD)의 Bounded Context 단위로 폴더를 나눕니다.

## 도메인 폴더 구조 (각 도메인 공통)

```
02-domains/{domain-name}/
├── CLAUDE.md             # 이 도메인의 개요, 진입점
├── domain-model.md       # 엔티티, 값 객체, 애그리게이트
├── business-rules.md     # 비즈니스 규칙, 불변 조건
├── workflows/            # 주요 유스케이스의 흐름
│   └── *.md
├── events.md             # 도메인 이벤트 목록
└── edge-cases.md         # 예외 상황 처리 정책
```

## Bounded Context 분리 원칙

### 좋은 분리의 신호
- 다른 팀이 독립적으로 변경 가능
- 한 도메인 안의 용어가 일관됨 (Ubiquitous Language)
- 트랜잭션 경계가 도메인 안에서 닫힘

### 나쁜 분리의 신호
- 한 유스케이스 처리에 5개 도메인을 호출해야 함 → 경계 재검토
- 같은 개념이 도메인마다 다른 의미 → 용어집에 redirect 명시

## 추천 도메인 목록 (예시)

| 도메인 | 책임 |
|---|---|
| `identity` | 인증, 권한, 사용자 프로필 |
| `payment` | 결제, 환불, 정산 |
| `catalog` | 상품, 카테고리, 검색 |
| `order` | 주문, 배송 추적 |
| `notification` | 이메일, 푸시, SMS |
| `analytics` | 이벤트 수집, 리포팅 |

## 도메인 간 통신
- **동기**: REST/gRPC (지연 민감, 강한 일관성)
- **비동기**: 이벤트 (느슨한 결합, 최종 일관성)
- 원칙: **기본은 비동기**, 동기는 정당화 필요
