# RFC 폴더 가이드

이 폴더는 **진행 중인 변경 제안**을 다룹니다.

## ADR vs RFC
- **RFC**: 결정 *전* 토론용. 변경 가능. 확정되면 ADR로 옮김.
- **ADR**: 결정 *후* 영구 기록. 불변.

## 작성 시점
- 큰 변경을 제안하고 싶을 때
- 코드보다 토론이 먼저 필요할 때
- 여러 팀의 합의가 필요할 때

## RFC 라이프사이클
```
draft → review → accepted → ADR로 변환 + 폴더에서 제거
                ↓
            rejected → 폴더에 보관 (이유 명시)
```

## 명명
```
RFC-001-event-sourcing-for-payments.md
RFC-002-graphql-adoption.md
```

## 좋은 RFC의 조건
1. **문제 정의**가 해결책보다 먼저
2. **여러 대안** 제시
3. **트레이드오프** 명시
4. **마이그레이션 계획** 포함 (큰 변경의 경우)
5. **결정 기한** 명시

## RFC가 필요 없는 경우
- 명확한 버그 수정
- 기존 패턴 따르는 작은 변경
- 한 사람만 영향받는 결정
